# NEOUL (너울) — 개념 정의서

> 이 문서는 Claude Code가 **너울(Neoul)** 프로젝트의 정체성을 이해하기 위한 개념 기준 문서다.
> 코드 작업 전에 이 문서를 먼저 읽고, "지금 만드는 것이 Geo-Intel Map인가 너울인가"를 항상 구분하라.
>
> ⚠️ **현재 상태 (2026-06-07): 개념 확정 단계. 코드 미착수.**
> Geo-Intel Map 엔진이 아직 Phase 8(박사 수준 추론) 진행 중이다.
> 너울의 실질 착수(퍼블리싱·구독·글→지도 변환)는 엔진 안정화 후
> "글 하나를 지도로 변환하는 데모"가 작동하는 시점을 분기점으로 한다.
> 지금은 이 문서를 **개념을 못 박아두는 용도**로만 둔다.

---

## 0. 한 줄 정의

> **너울 = 지정학 분석글을 인터랙티브 지도로 *읽는* 퍼블리싱 플랫폼.**
> 글을 지도에 올리면, 그 글이 다루는 국가·물동량·해상 흐름이 지도 위에서 살아 움직인다.

```
한국명   너울
영문명   Neoul
도메인   neoul.co
포지션   독자 중심 — "지도로 읽는 지정학 콘텐츠"
태그라인 세계가 출렁이는 방식을 지도로 읽다 / Read the world as it moves
```

---

## 1. 이름의 세 레이어

이름 하나에 세 가지 의미가 겹쳐 있다. 이건 우연한 작명이 아니라 프로젝트의 본질과 맞물린다.

| 레이어 | 의미 | 프로젝트 연결 |
|--------|------|--------------|
| **너울** | 큰 바다 물결, 해류, 해상 흐름 | 핵심 콘텐츠 — 물동량·SLOC·지정학적 흐름 |
| **Neo** | 새로운, 신흥(emerging) | 신흥 안보(emerging security)의 파동 |
| **Wave** | 파동이 퍼져나간다 | cascade(연쇄) 분석의 은유 — 한 사건이 다른 사건으로 전파 |

---

## 2. 가장 중요한 구분: 너울 ≠ Geo-Intel Map

**너울은 Geo-Intel Map이 아니다. 너울은 Geo-Intel Map *위에* 세워지는 별개의 제품이다.**

이 둘을 혼동하면 안 된다. 기술 스택은 공유하지만, 정체성·사용자·목적·UX 철학이 전부 다르다.

### 2.1 핵심 대조표

| 축 | **Geo-Intel Map (기존)** | **너울 (신규)** |
|----|--------------------------|------------------|
| **레이어** | 연구·개발 레이어 (엔진) | 서비스·제품 레이어 |
| **한 문장** | "추론하는 지도" — 분석을 *생성*하는 도구 | "읽는 지도" — 분석을 *게시·소비*하는 플랫폼 |
| **목적** | 유민 개인의 정치외교학 학습 + 인과 추론 | 다수 독자가 국제정치를 눈으로 이해 |
| **주체/사용자** | 1인 (유민 본인 = 분석가) | 다수 (작가 + 독자) |
| **핵심 자산** | IA-Engine, 8단계 추론, cascade, 이론 라이브러리 | 글→지도 변환 UX, 퍼블리싱, 독자 경험 |
| **산출물** | 인사이트 카드, 증거 등급, 추론 사다리 등급 | 게시된 분석글 + 인터랙티브 지도 뷰 |
| **워크플로우** | 쿼리 → 엔진이 인사이트 생성 → 평가 → 수정 (평가-구현 루프) | 작가가 글 게시 → 글이 지도 레이어로 변환 → 독자가 지도로 읽음 |
| **성공 지표** | 신뢰도 점수, Granger 검증, 박사 수준 정합성 | 게시글 수, 독자 수, 글→지도 변환 정확도, 구독 전환 |
| **공개 여부** | OSS (GitHub 공개 가능) | 상업 서비스 (별도 레포·브랜드) |

### 2.2 결정적 차이 — 방향이 반대다

Geo-Intel Map의 시스템 프롬프트는 이렇게 선언한다:

> *"이 시스템은 '보는 지도'가 아니라 '추론하는 지도'를 지향한다."*

너울은 그 반대 방향으로 한 걸음 더 간다. **너울은 추론 엔진을 백엔드에 두되, 전면에는 인간 작가가 쓴 글을 놓고, 그 글을 다시 독자가 "보는 지도"로 되돌려준다.**

```
Geo-Intel Map:  데이터 → [추론 엔진] → 인사이트 (분석가 1인이 읽음)
너울:           인간이 쓴 분석글 → [변환] → 지도 레이어 (다수 독자가 봄)
                                      ↑
                          Geo-Intel Map 엔진이 글 평가·검증·데이터 연결을 담당
```

핵심: **너울에서 Geo-Intel Map은 "콘텐츠 생산자"가 아니라 "글을 검증하고 지도화하는 엔진"으로 역할이 바뀐다.**

## 3. 관계 구조 — 같은 기술, 다른 브랜드

```
geo-intel-map (엔진 / OSS / 별도 레포)
    │
    │  엔진·라이브러리·데이터 파이프라인으로 활용
    ▼
neoul (서비스 / 상업 / 별도 레포·브랜드)
    ├─ 글 퍼블리싱 (CMS)
    ├─ 글 → 지도 변환
    ├─ 독자용 지도 뷰 UX
    └─ 계정·구독·결제
```

**원칙: 기술은 Geo-Intel Map 위에, 브랜드와 레포는 분리한다.**

분리 이유:
- OSS로 공개한 엔진에 수익 모델을 직접 얹으면 라이선스 충돌 가능성
- 개발자/분석가용 도구와 일반 독자용 서비스는 UX 철학이 근본적으로 다름
- 두 제품의 성공 지표·사용자가 완전히 다름

## 4. 무엇을 재사용하고, 무엇을 새로 만드는가

### 4.1 Geo-Intel Map에서 재사용 (이미 있는 자산)

| 자산 | 너울에서의 새 역할 |
|------|---------------------|
| Leaflet 지도 + 레이어 시스템 | 독자용 지도 뷰의 기반 |
| ACLED·GDELT·FRED·EIA·NASA FIRMS 등 데이터 파이프라인 | 글에 연결될 실시간 데이터 백본 |
| Cascade 엔진 + 룰북 | 글이 다루는 사건의 연쇄를 지도에 시각화 |
| IA-Engine (신뢰도 점수·H1·Granger) | **글을 게시 전 평가·검증하는 게이트** |
| 이론 라이브러리 (40+ IR 이론) | 글에 자동으로 이론 태그·맥락 부여 |
| 7대 축 메타데이터 / Event 모델 | 글에서 추출한 엔티티의 정규화 스키마 |

### 4.2 너울에서 새로 만들어야 하는 것 (신규)

| 신규 컴포넌트 | 설명 | 기술 난관 |
|---------------|------|----------|
| **글 → 지도 변환 메커니즘** | 텍스트에서 국가·해역·항구 추출(NER) + 관계 추출 → 지도 레이어 생성 | ★★★ 최대 난관 |
| **퍼블리싱 CMS** | 글 작성·게시·편집 | ★★ |
| **독자용 지도 뷰 UX** | 글을 읽으며 지도가 연동되는 경험 | ★★ |
| **계정·인증** | 작가/독자 구분 | ★★ |
| **구독·결제** | 수익 모델 | ★★ |

> 글→지도 변환의 두 방향:
> - **A) 자동 추출** — AI가 글을 읽고 자동으로 지도 데이터 생성 (스케일 가능, 정확도 낮음)
> - **B) 반자동 태깅** — 작가가 글 쓸 때 국가/이벤트 태그 직접 지정 (정확, 작업 부담)
> 둘 중 무엇을 우선할지는 MVP 설계 시 결정한다.

## 5. 시장 포지션과 차별화

너울이 채우는 시장 공백:

| 기존 서비스 | 부족한 것 | 너울이 채우는 것 |
|-------------|-----------|------------------|
| Stratfor 등 엔터프라이즈 인텔 | 가격이 높고 일반인 접근 불가 | 저가·접근성 |
| Substack 지정학 뉴스레터 | 텍스트만, 지도 없음 | 글 + 지도 연동 경험 |
| Conflict Mapping 류 지도 도구 | 분석글 없음, 도구만 | 콘텐츠 + 시각화 통합 |
| Mapdis 류 AI 지도 | UGC 없음, AI 생성 콘텐츠뿐 | **인간 분석가의 글 기반** |

**핵심 차별화: "분석글을 지도로 읽는다"는 경험 자체가 현재 시장에 없다.** 이게 너울의 자리다.

주의: 포지션을 "지정학 정보 플랫폼"으로 넓게 잡으면 Stratfor와 싸우게 된다. **"분석글 → 지도 변환 경험"**이라는 좁고 명확한 포지션을 지킨다.

## 6. 개발 로드맵에서 너울의 위치

```
1. 인사이트 분석 엔진 완료        ← 이것이 곧 Geo-Intel Map (현재 진행 중 — Phase 8 박사 수준 추론)
2. 글 변환 메커니즘 (글 형식·말투 조정 포함)
3. 글 게시 웹사이트
4. 지도 연동 기능
5. UI/UX 개선
6. 타인이 사용하는 플랫폼화
```

**현재 위치: 1단계(Geo-Intel Map 엔진)가 아직 진행 중.** 너울의 실질 착수(2단계 이후)는 엔진이 안정화된 뒤다. 지금 단계에서 너울의 퍼블리싱·구독 껍데기를 먼저 만드는 것은 순서가 거꾸로다.

분기점: **Geo-Intel Map 엔진 위에서 "글 하나를 지도로 변환하는 데모"가 작동하는 시점**이 너울을 별도 레포·브랜드로 분리하는 지점이다. (Geo-Intel Map 로드맵 기준 Phase 9 = 시각화 착수 지점과 일치)

## 8. 엔진 단계에서의 전방 호환 원칙 (지금 지킬 것 — 단 하나)

너울 코드는 지금 만들지 않는다. 그러나 엔진이 생산하는 인사이트 출력물이
나중에 "글의 씨앗"이 될 수 있도록 다음 하나만 유지한다:

> **인사이트 출력은 "글의 씨앗" 품질을 유지한다** — 인간 작가가 발행 가능한 글로
> 확장할 수 있을 만큼 구조화되고, 엔티티(국가·초크포인트·이벤트)가 깨끗이 추출
> 가능해야 한다. (이미 Event 모델 + 7대 축 메타데이터가 이 역할을 수행 — 추가 비용 없음.)

이 원칙은 비용이 0이며, 나중 재작업을 막는다. 그 외 CMS·구독·발행 레이어는 절대 지금 만들지 않는다.
