# NotebookLM PPT 생성 프롬프트

> 사용법: NotebookLM에 **`presentation_russia_ukraine_slides.md` 하나만** 소스로 업로드한 뒤, 아래 프롬프트를 복사해 입력한다.
> (원본 `presentation_russia_ukraine.md`는 대본·메모가 섞여 있으니 **업로드하지 말 것.**)

---

## 복사용 프롬프트

```
이 소스는 학부 세미나 발표용 슬라이드 원고입니다. 이 내용으로 발표 슬라이드 덱을 만들어 주세요.

[구조 규칙]
- "## 슬라이드 N" 제목 하나당 정확히 1장의 슬라이드로 만드세요. 슬라이드를 합치거나 쪼개지 마세요.
- 슬라이드 순서와 번호(1, 2, 2-A, 3, 4-1 … 11)를 원본 그대로 유지하세요.
- 슬라이드 제목은 "—" 뒤의 문구를 쓰세요.
- 불릿과 표는 그대로 슬라이드 본문으로 옮기되, 너무 길면 핵심만 간결하게 다듬으세요.

[내용 규칙 — 매우 중요]
- 소스에 없는 내용을 새로 지어내지 마세요(환각 금지). 수치·날짜·출처를 바꾸지 마세요.
- 다음 정직성 표기는 절대 삭제하지 마세요: 추론 등급([상관]·[기술적/구조적]), [UNVERIFIED], "📎 출처:", "한계", "혼동 주의", "솔직한 고지".
- 표의 판정 기호(❌ 열세 / ⚠️ 부분·불확실)와 Granger 수치(p값·등급)는 그대로 두세요.

[형식]
- 언어: 한국어. 톤: 학술 발표(차분하고 명료하게).
- 슬라이드당 텍스트는 핵심 위주로. 문장형 설명은 짧은 명사구 불릿으로 바꿔도 됩니다.
- "📎 출처:" 줄은 해당 슬라이드 하단에 작은 글씨로 배치하세요.

[제외]
- 파일 맨 위의 안내 블록(⚠️로 시작하는 사용 규칙)은 슬라이드로 만들지 마세요.
```

## 보조 팁

- **발표자 노트가 필요하면:** 위 프롬프트 끝에 한 줄 추가 —
  `각 슬라이드의 발표자 노트(speaker notes)는 비워 두세요. 노트는 발표자가 따로 작성합니다.`
  (대본은 원본 `presentation_russia_ukraine.md`의 `[대본]` 부분에 있습니다. 슬라이드 덱이 나오면 그걸 노트로 옮기세요.)
- **도식 슬라이드(2-A·3·8):** 이미 불릿으로 평문화돼 있어 그대로 잘 변환됩니다. 더 보기 좋게 하려면 생성 후 "슬라이드 2-A를 좌→우 플로우 다이어그램으로 다시 그려줘"처럼 개별 요청.
- **구조가 뭉개지면:** NotebookLM은 재요약 경향이 있어 5박자 구조가 흐려질 수 있습니다. 이때는 **Gamma**(마크다운 붙여넣기 → "각 ## 를 슬라이드로") 또는 **Marp/Slidev**(제목 기반 변환)가 구조 보존에 더 정확합니다.
- **분량:** 총 16장(슬1·2·2-A·3·4-1~4-3·5-1~5-3·6·7·8·9·10·11) + 마무리 1장. 13분 분석 기준 적정.
